# Детальное сравнение производительности ONNX Runtime vs NCNN

## Тестовая конфигурация

- **Устройство**: Raspberry Pi
- **Модель**: lp_regressor (512x288 вход, 4 угла на выход)
- **Потоков**: 4
- **Изображений**: 200
- **Обработано**: 193

## Результаты

### ONNX Runtime

| Метрика | Значение |
|---------|----------|
| Среднее время инференции | **106.2 ms** |
| Минимальное время | 90.5 ms |
| Максимальное время | 330.7 ms |
| Общее время обработки | 36.60 сек |
| FPS (теоретический) | ~9.4 |

### NCNN

| Метрика | Значение |
|---------|----------|
| Среднее время инференции | **113.1 ms** |
| Минимальное время | 78.1 ms |
| Максимальное время | 289.6 ms |
| Общее время обработки | 40.33 сек |
| FPS (теоретический) | ~8.8 |

## Сравнение

- **ONNX Runtime быстрее на 6.5%** (6.9 ms разница)
- **Разница**: минимальная, оба формата показывают сопоставимую производительность
- **Стабильность**: NCNN показывает более стабильные результаты (меньший разброс)

## Размеры моделей

- **ONNX**: 5.7 MB
- **NCNN**: 2.9 MB (**в 2 раза меньше!**)

## Выводы

1. **Производительность**: ONNX Runtime немного быстрее на больших объемах данных
2. **Размер модели**: NCNN значительно компактнее (2.9MB vs 5.7MB)
3. **Стабильность**: NCNN показывает более предсказуемое время выполнения
4. **Рекомендация**: 
   - Для максимальной скорости: **ONNX Runtime**
   - Для экономии памяти: **NCNN**

## Примечания

- Первые запуски могут быть медленнее из-за инициализации
- Результаты могут варьироваться в зависимости от сложности изображений
- Оба формата готовы к production использованию
