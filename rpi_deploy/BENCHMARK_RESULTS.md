# Результаты тестирования производительности

## Конвертация модели

Модель `lp_regressor.onnx` успешно конвертирована в NCNN формат с использованием **pnnx**.

### Размеры моделей:
- **ONNX**: 5.7MB
- **NCNN**: 2.9MB (почти в 2 раза меньше!)

## Производительность

### Тестовая конфигурация:
- Устройство: Raspberry Pi
- Модель: lp_regressor (512x288 вход, 4 угла на выход)
- Потоки: 4

### Результаты:

#### ONNX Runtime:
- Инференция: ~100-110ms
- Общее время: ~170-200ms

#### NCNN:
- Инференция: ~94-235ms (первый запуск медленнее)
- Общее время: ~180-300ms

## Выводы

1. **Размер модели**: NCNN модель почти в 2 раза меньше (2.9MB vs 5.7MB)
2. **Скорость инференции**: Сопоставима с ONNX Runtime
3. **Первый запуск**: NCNN может быть медленнее из-за инициализации
4. **Стабильность**: Оба формата работают стабильно

## Рекомендации

- **ONNX Runtime**: Проще в использовании, хорошо документирован
- **NCNN**: Меньший размер модели, оптимизирован для ARM процессоров

Для данной модели оба формата показывают сопоставимую производительность.
